# IS-001 — Session Bootstrap

1. Read `docs/CONTEXT_INDEX.md`.
2. Identify task type → select **Role** + **IS** from matrix.
3. Read role charter: `docs/roles/<ROLE>.md`.
4. Load only files listed in matrix **Load** column for this task type.
5. If `.session-state.yaml` exists in project root, read it for prior session context.
6. State (in thinking): role, IS, files loaded, invariants at risk.
7. Do NOT read conversation history or README unless matrix says so.
8. If task type unknown → adopt ARCHITECT role, run IS-001, produce plan before coding.
9. Proceed to task-specific IS (IS-002..IS-005) or work directly if scope is clear.
