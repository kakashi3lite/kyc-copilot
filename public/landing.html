How it works

Three steps.
One evidence-backed dossier.

The entire KYC pipeline runs as a deterministic state machine. Every decision is traceable. Every claim is sourced.

Step 01
📋
Submit a case

One API call with company name, registration number, and jurisdiction. Or use the dashboard to submit manually. That's the entire input.

Step 02
⚡
Agent investigates

The 6-node LangGraph pipeline queries APIs, falls back to browser scraping for regional registries, then audits every claim against the evidence ledger via the guardrail node.

Step 03
📄
Receive the dossier

A 3-paragraph AMLD6-aligned EDD narrative lands in your dashboard — every factual claim carrying a [Source: KEY] citation that maps to a deterministic API payload or screenshot.

Evidence Ledger

Every claim.
Provably sourced.

EU regulators increasingly require a documented evidence trail for every CDD/EDD finding. KYC Copilot's immutable ledger maps each [Source: KEY] citation to a deterministic API response or timestamped screenshot — auditor-ready on demand.

🔗
Append-only ledger
Evidence entries are never deleted or modified. Chain-of-custody is preserved end-to-end.
🛡
Guardrail auditor
A second LLM scans the draft dossier and strips any claim that lacks a valid evidence key before it reaches you.
📥
One-click AMLD6 report
Download a structured JSON report referencing the exact AMLD6 article that governs the risk level — ready for NCA submission.
👤
Human-in-the-Loop
High-risk cases pause the graph and wait for senior compliance officer sign-off before finalising — never auto-approved when it matters.
